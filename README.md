# Civic Accessibility Issue Tracker (CivicBot System)

## 📌 Overview
The Civic Accessibility Issue Tracker is a full-stack web application designed to help residents report, track, and manage municipal accessibility issues. The system integrates AI-powered insights and a chatbot (CivicBot) to enhance issue understanding, prioritization, and decision-making.

## 🚀 Features

### 🟢 Issue Reporting & Tracking
The system allows users to report and monitor civic accessibility issues efficiently.

**Reporting Features:**
- Users can submit issues with:
  - Title
  - Description
  - Location
  - Category
  - Priority
- Optional image upload support
- Automatic timestamping (`createdAt`, `updatedAt`)

**Tracking Features:**
- Users can view all reported issues in a dashboard
- Each issue displays:
  - Current status (Open, In Review, In Progress, Resolved)
  - Assigned municipal team
  - Priority level
  - AI-generated summary
- Issues are sorted by most recent activity

### 🔔 Alerts & Notifications
- Notifications are generated by the backend system
- Triggered when:
  - Issue is created
  - Issue is assigned
  - Status is updated
  - Issue is resolved
- Delivered to user dashboard

### 🤖 AI-Powered Issue Insights
- AI Summary
- AI Category Suggestion
- AI Priority Reason

### 💬 CivicBot Chatbot
- Answers questions
- Provides guidance
- Uses real issue data

### 🏢 Issue Assignment System
Teams:
- Accessibility Team
- Transit Accessibility Unit
- Municipal Maintenance
- Traffic Signals
- Road Safety Team

### 👍 Community Advocate Feature
- Upvotes system
- Tracks supporters

## 🏗️ Tech Stack
Frontend: React, TypeScript, Tailwind  
Backend: Node.js, Express, GraphQL  
Database: MongoDB  
AI: Google Gemini  

## ⚙️ Setup

Backend:
cd backend
npm install

.env:
MONGO_URI=your_mongo
GOOGLE_API_KEY=your_key

npm run dev

Frontend:
cd frontend
npm install
npm run dev
